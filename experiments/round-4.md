# Round 4 Experiment: Cached LLM Direct Rewrites

## Goal

Test whether direct LLM-generated JavaScript rewrites can find useful alternatives that deterministic AST rewrite rules missed on real-project artifacts.

## Method

Round 4 treats the LLM as an untrusted candidate generator:

1. Select small source windows from Round 2 real-project artifacts.
2. Ask for shorter equivalent JavaScript for that complete window.
3. Cache the generated responses in `benchmarks/round4/llm-candidates.json`.
4. Splice each response into the original artifact only if the source-window SHA-256 still matches.
5. Minify the full artifact with Terser.
6. Verify the minified output parses.
7. Score candidates by Brotli size first, then gzip, then raw bytes.

## Validation Status

This round does not run upstream library test suites. It verifies source-window identity, Terser minifiability, and post-minify syntax. Behavioral validation is required before any rewrite can be considered production-safe.

## First Run

Command:

```sh
npm run experiment:round4
```

Results are written to `results/round4.json` and `results/round4.md`.

The first run uses cached LLM responses because no live `OPENAI_API_KEY` is configured in this environment.

## Experiment Log

- Date: 2026-05-01.
- Candidate source: cached LLM responses in `benchmarks/round4/llm-candidates.json`.
- Corpus: six source windows from `react`, `moment`, and `vue`.
- Validator: source-window SHA-256 match, Terser minifiability, and Babel parseability after minification.
- Selector: smallest Brotli output, then gzip, then raw bytes.

## Observations

- All 6 cached LLM candidates were syntax-valid after splicing and Terser.
- 3 candidates were accepted by the Brotli-first selector.
- Aggregate accepted-window delta was 21 raw bytes smaller, 4 gzip bytes smaller, and 68 Brotli bytes smaller than the corresponding Terser baselines.
- The `moment-isObjectEmpty-early-fallback` candidate is a useful example of why Brotli-first scoring matters: it was 2 raw bytes smaller and 57 Brotli bytes smaller, but 2 gzip bytes larger.
- The `react-getIteratorFn-single-return`, `vue-makeMap-shared-return`, and `vue-looseIndexOf-one-line` candidates were rejected by the selector because Terser baseline was better or tied under the Brotli-first score.

## Limitations

- This is a tiny sample, not a full-corpus run.
- The candidates were cached from this Codex session rather than generated by a live API call inside the runner.
- No upstream behavioral tests were executed, so accepted candidates are compression signals rather than correctness proof.

## Next Action

Add live model generation when an API key is available, expand the window sampler, and run upstream behavioral or differential tests before treating any direct rewrite as safe.
